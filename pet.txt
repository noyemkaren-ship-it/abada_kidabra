Что это такое
Сайт для издателя, где можно хранить книги и управлять ими.

Что делает пользователь
Создает книгу — вводит название, автора, год, цену, выбирает категорию

Смотрит список книг — видит все книги с фильтрацией и поиском

Редактирует книгу — меняет цену, описание, год

Удаляет книгу — убирает из базы

Управляет категориями — создает и удаляет категории (роман, фантастика и т.д.)

Как это работает
Пользователь отправляет запрос через API (или через сайт)

FastAPI принимает запрос, проверяет данные через Pydantic

SQLAlchemy сохраняет или достает данные из PostgreSQL

Ответ возвращается пользователю в формате JSON

Главные сущности
Сущность	Что хранит	Пример
Книга	Название, автор, год, цена, описание	"1984", Оруэлл, 1949, 800 руб.
Категория	Название, описание	"Антиутопия", "Роман"
Что можно делать с книгами
Действие	Как
Создать	POST /book — передать данные
Найти все	GET /books — получить список
Найти одну	GET /books/{id} — получить по номеру
Изменить	PUT /books/{id} — обновить данные
Удалить	DELETE /books/{id} — стереть из базы
Поискать	GET /books?search=Оруэлл — найти по автору или названию
Отфильтровать	GET /books?min_price=500&max_price=1000 — книги в ценовом диапазоне
Отсортировать	GET /books?sort=price&order=desc — от дорогих к дешевым
Что можно делать с категориями
Действие	Как
Создать	POST /categories
Посмотреть все	GET /categories
Удалить	DELETE /categories/{id}
Связи между данными
Одна категория → много книг

У каждой книги → одна категория

Автор — просто текст внутри книги, отдельной таблицы нет

Технологии которые ты используешь
Технология	Зачем
FastAPI	Создаешь API-эндпоинты
SQLAlchemy	Общаешься с базой данных
Pydantic	Проверяешь данные (год не 3000, цена не -500)
PostgreSQL	Хранишь книги и категории
Alembic	Обновляешь структуру таблиц
Uvicorn	Запускаешь сервер
Итоговый результат

Добавлять книги с автором, годом, ценой

Искать книги по автору или названию

Фильтровать по цене и году

Сортировать и листать страницы

Управлять категориями
